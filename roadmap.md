# 🗺️ 시스템 엔지니어 트레이닝 로드맵

> **목표**: 2027년 6월 21일까지 13개 트랙 완주 → 정규직 시스템 엔지니어 취업
> **최종 수정**: 2026-07-20
> **자격증 병행**: CCNA → RHCSA → AWS SAA

## 📌 진행 방식 요약

| 구분 | 순서 |
|---|---|
| 평일 (회사, 짬짬이) | 컴퓨터구조 → 네트워크 → 리눅스 → 윈도우 → 보안 → 모니터링&백업 |
| 주말 | Git(선행) → HTML/CSS/JS + Java/Spring/SpringBoot + SQL (병행) → 자동화 → AWS → 가상화/컨테이너 |

> 💡 평일 순서는 CCNA 자격증 공부 순서(네트워크 먼저)와 맞물려, 회사에서는 컴퓨터구조를 먼저 진행하도록 재배치함 (콘텐츠 중복 방지 + 학습 리듬 확보)

## Track 1. 네트워크

**목표**: 암기식 이론이 아니라, 실제 장애 상황을 진단하고 원인을 계층별로 좁혀나갈 수 있는 감각을 기른다.
> "OSI 7계층이 뭐다" 수준이 아니라 "지금 이 증상이 어느 계층 문제인지 파악하고, 어떤 명령어로 확인/조치할지 즉시 판단"이 목표.

**우선순위**: 1순위 (평일 짬짬이, 매일 진행)

### 학습 항목
- [ ] 1. 기초 개념 재정립 — OSI 7계층, TCP/IP 4계층, MAC/IP/포트 개념 (`장애 시 이 계층에서 뭘 확인하나`와 매칭해서 암기)
- [ ] 2. IP / 서브넷 / 라우팅 기초 — 서브넷 마스크 계산, 사설/공인 IP, NAT, 게이트웨이
- [ ] 3. 네트워크 장비 이해 (스위치/L2, 라우터/L3, AP) — VLAN, 포트 상태 확인, 물리 연결 점검
- [ ] 4. 무선 네트워크(Wi-Fi) 기초 — 2.4GHz/5GHz, 채널/간섭, WPA2/3
- [ ] 5. DNS — nslookup/dig, DNS 캐시, hosts 파일, 사내 DNS 이슈 진단
- [ ] 6. DHCP — 임대 갱신, 충돌, 사내 DHCP 서버 로그 확인
- [ ] 7. 방화벽 / NAT / 포트포워딩 — 포트 차단 확인, 방화벽 정책 진단
- [ ] 8. 트러블슈팅 도구/명령어 실전 (통합 복습) — `ping`, `tracert/traceroute`, `ipconfig/ifconfig`, `netstat`, `nslookup`, `telnet/nc`, `arp -a`, `pathping`
- [ ] 9. 프로토콜 심화 — HTTP/HTTPS, TLS 기본 동작, TCP 3-way handshake, UDP 차이
- [ ] 10. VPN 기초 — 사내 원격 접속(VPN 터널) 구조, 흔한 접속 실패 원인
- [ ] 11. 패킷 캡처 기초 — Wireshark 기본 사용법

**실무 적용**: 매일 1개 [현업 실무 시나리오 문제] → 이론 → 문제 → CLI 명령어 조합 도출
사고 흐름: `PC 자체 → 스위치/AP → 사내망 → 외부 회선(ISP)` 순으로 좁혀나가는 흐름 반복 훈련

## Track 2. 컴퓨터 구조

**목표**: 학부 수준의 이론 암기가 아니라, 서버/PC 하드웨어 이상 징후를 진단하고 OS/성능 트러블슈팅의 근본 원리를 이해하는 실전 감각을 기른다.

**우선순위**: 1순위급 (평일 짬짬이 진행 — 회사 내 실제 진행 순서 1번)

### 학습 항목
- [ ] 1. 컴퓨터 구성 기본 — CPU/메모리/스토리지/메인보드/전원부 역할과 상호작용, 병목(bottleneck) 개념
- [ ] 2. CPU 구조 기초 — 코어/스레드, 클럭 속도, 캐시(L1/L2/L3), 멀티코어 vs 멀티스레딩
- [ ] 3. 메모리(RAM) 구조 — 물리 메모리 vs 가상 메모리, 페이징/스와핑
- [ ] 4. 스토리지 구조 — HDD vs SSD, RAID 0/1/5/10, IOPS/처리량
- [ ] 5. 버스 & I/O 구조 `(선택 항목)` — 시스템 버스, PCIe, 컨트롤러 연결 구조
- [ ] 6. 프로세스/스레드 실행 원리 — 컨텍스트 스위칭, 인터럽트 처리, 스케줄링
- [ ] 7. 부팅 과정의 하드웨어적 이해 — POST, BIOS/UEFI, 비프음/POST 코드 진단
- [ ] 8. 네트워크 하드웨어 기초 — NIC 동작 원리, MAC 주소
- [ ] 9. 서버 하드웨어 특화 지식 — ECC 메모리, 서버용 CPU/RAID 카드, 핫스왑, 이중화
- [ ] 10. 성능 진단 실전 연결 — "느리다" 증상을 CPU/메모리/디스크/네트워크로 구조적으로 좁혀나가기
- [ ] 11. 하드웨어 진단 실전 도구 종합
  - Windows: `msinfo32`, 장치 관리자
  - macOS: `system_profiler SPHardwareDataType`, Activity Monitor
  - 디스크 상태: `smartctl` (Windows/macOS 공통, smartmontools 설치)
  - 메모리 진단: `memtest86` 개념 (실습은 선택)
  - 서버 원격 관리(IPMI/iDRAC/iLO) `(개념 이해 위주)` — 물리 접근 없이 상태 확인하는 용도

**실무 적용**: 매일 1개 [현업 실무 시나리오 문제] → 이론 → 실전 문제 → 진단 절차/명령어 조합 도출

## Track 3. 리눅스 운영체제

**목표**: 이론 암기가 아니라, 서버를 설계·구축하고 운영 중 상태를 진단하고 문제를 해결할 수 있는 실전 감각을 기른다.

**우선순위**: 1순위 (평일 짬짬이, 매일 진행)

- [ ] 5. 패키지 관리 — `yum/dnf` (RHEL 계열), `apt` (Debian 계열), 의존성 문제 해결
- [ ] 6. 디스크/스토리지 설계 & 관리 — 파티션 설계, LVM, `df`, `du`, `fdisk/lsblk`, `mount/umount`

- [ ] 9. 서비스/데몬 관리 — `systemctl`(서비스/부팅 관리), 서비스 자동시작 설정
- [ ] 10. 사용자/그룹 관리 — `useradd/usermod`, `/etc/passwd`, `/etc/shadow`, sudo, 계정 잠금/해제
- [ ] 11. 크론(Cron) & 스케줄링 — `crontab`, `at`, 배치 작업 실패 시 로그 확인
- [ ] 12. 쉘 스크립트 기초 문법 `(선행 학습)` — 변수, 조건문, 반복문, 함수
- [ ] 13. 성능 진단 도구 — `vmstat`, `iostat`, `sar`, `free -h`
- [ ] 14. 부팅 프로세스 & 트러블슈팅 — GRUB, runlevel/systemd target, 부팅 실패 복구
- [ ] 15. SSH & 원격 접속 관리 — `ssh-keygen`, `known_hosts`, `sshd_config` 보안 설정

**실무 적용**: 매일 1개 [현업 실무 시나리오 문제] → 이론 → 실전 문제 → 명령어 조합 도출

- [ ] 13. 스케줄링 & 배포 — crontab/작업 스케줄러, venv 관리, `requirements.txt`
- [ ] 14. Windows 자동화 (PowerShell 연계) — Python-PowerShell 역할 분담, `pywin32`
